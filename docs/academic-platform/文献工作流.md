# 文献工作流

## 目标
这套工作流的目标不是“快速堆摘要”，而是把每条文献都变成可核验、可引用、可回溯的证据。

## 1. 中文文献如何检索
### 主路线
1. 通过学校图书馆入口或正规访问方式打开 CNKI
2. 优先检索：
   - 软件工程毕业设计 / 毕业论文方法论
   - 与你的具体题目相关的系统设计、需求分析、测试、实现案例
   - 中文标准、规范、行业研究或应用论文
3. 初筛时只保留和题目直接相关、近五年优先、来源明确的文献

### 入库前先做两件事
1. 看题名和摘要是否真与你的题目相关
2. 看来源是否足够正规，例如核心期刊、学报、学位论文、正式会议论文

## 2. 如何通过 Zotero Connector 或人工方式入库
### Connector 导入
1. 打开 CNKI 文献详情页
2. 保持 Zotero Desktop 已启动
3. 点击浏览器中的 Zotero Connector
4. 导入后立刻核对条目元数据

### 人工导入
以下场景直接改用人工导入：

- Connector 没识别出条目
- CNKI 页面被验证码或登录状态打断
- 页面结构异常，导入字段缺失

人工导入时必须至少保证这些字段正确：

- 题名
- 作者
- 年份
- 来源
- 卷期页码
- DOI（若有）
- 原始链接

## 3. 如何整理到 Zotero 与 Markdown 卡片
### Zotero 里做什么
- 负责条目管理
- 负责 PDF 附件管理
- 负责标签、集合、导出

### Markdown 卡片里做什么
- 记录真正要进入论文的“事实”
- 记录该文献可用于哪一章
- 记录具体页码或出处位置

建议流程：

1. 文献入 Zotero
2. 判断归属集合
3. 下载或挂接 PDF
4. 新建一张文献事实卡片
5. 只记录有出处、能定位的事实点

## 4. 如何对外文文献做真实性核验
外文文献不要只看标题像真的就引用。

推荐流程：

1. 优先找 DOI
2. 用 `scripts/verify_reference.py` 做 Crossref 核验
3. 至少核对：
   - 标题
   - 作者
   - 年份
   - DOI
   - 期刊 / 会议名称
4. 只有核验基本一致后，才进入 Zotero 和事实卡片

注意：

- arXiv 预印本、课程讲义、网页博客、技术报告不一定都在 Crossref 里
- Crossref 没命中，不等于文献一定是假的
- 但 Crossref 没命中时，必须显式标为“待人工复核”

## 5. 如何把 PDF 转为结构化材料
### 首选：Doc2X
如果你已经完成登录或拿到 `DOC2X_API_KEY`：

1. 上传真实 PDF
2. 解析为 Markdown / 结构化文本
3. 只提取与你论文直接有关的内容
4. 回填到文献事实卡片

### 当前保底方案
如果 Doc2X 还没正式接入：

1. 在 Zotero 中打开 PDF
2. 手工摘录摘要、方法、结论、图表说明
3. 必须记录页码或章节位置
4. 再写入事实卡片

这虽然没有 Doc2X 自动化高效，但真实、稳妥、低风险。

## 6. 如何把真实证据交给后续论文写作
后续进入论文写作阶段时，不要直接对着 PDF 或网页写正文，而是只对着“已核验的事实卡片”写。

建议规则：

1. 论文里的每个事实性句子，都应能回溯到至少 1 张事实卡片
2. 每张事实卡片都应能回溯到原始文献条目
3. 每个事实点都尽量带页码或出处定位

这样做的好处是：

- 不容易编造文献
- 不容易写空话
- 不容易把 AI 总结当成原文事实

## 7. 如何降低“空话、假文献、AI味”风险
执行时重点遵守下面几条：

- 中文论述优先引用真实 CNKI 文献、真实学位论文、真实标准
- 外文文献先过 Crossref 核验，再进入写作池
- 不引用来源模糊、检索不到、字段对不上的材料
- 不把模型总结当成原文证据
- 不把没有页码或具体出处的句子写进最终论文正文

## 8. 建议的最小闭环
你可以先跑通这个最小闭环：

1. 从 CNKI 找 3 篇中文文献
2. 成功入库 Zotero
3. 从开放来源补 2 篇外文文献
4. 用校验脚本核验这 2 篇外文文献
5. 为其中 2 到 3 篇文献各写 1 张事实卡片

只要这个闭环打通，平台就已经达到“可正式进入论文准备阶段”的状态。

## 9. 写作素材池的入池规则
后续论文写作时，不是所有材料都能直接拿来写正文。

只有下面四类材料可以进入写作素材池：

1. 已入 Zotero 的中文文献
2. 已通过 Crossref 基本核验的外文文献
3. 已完成事实卡片并带出处定位的中文或外文文献
4. 你自己的真实项目材料，例如代码、数据库结构、页面截图、接口说明、测试记录

以下材料禁止直接进入写作素材池：

1. 还没入 Zotero 的文献
2. 没有核验过的外文文献
3. 没有页码或出处定位的摘录
4. 没有原始来源支撑的模型总结

## 10. 进入正文写作前的最短流程
在开始写任何一章正文前，至少先满足下面条件：

1. 中文文献至少有 3 篇已入 Zotero
2. 外文文献至少有 2 篇完成核验
3. 至少有 2 到 3 张文献事实卡片已经填完
4. 已把与你项目直接相关的真实材料整理出来
5. 已明确每张卡片更适合支持哪一章

达到这 5 条后，再进入“论文目录设计与分章写作准备”才最稳。
