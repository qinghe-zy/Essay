# 外文文献校验说明

## 目的
这个说明对应 `scripts/verify_reference.py`。

它的目标不是替代人工判断，而是把最常见的“假外文文献、错 DOI、作者年份对不上”的问题提前筛掉。

## 校验范围
脚本目前基于 Crossref 做轻量校验，适合下面这类文献：

- 有 DOI 的期刊论文
- 有 DOI 的会议论文
- 可被 Crossref 收录的正式出版物

不适合或命中率不稳定的场景：

- arXiv 预印本
- 纯网页文章
- 一些图书章节
- 一些没有 DOI 的技术报告

## 支持核对的字段
脚本至少支持核对这些字段：

- 标题
- 作者
- 年份
- DOI
- 期刊 / 会议名称

## 使用方法
### 方式一：按 DOI 核验

```powershell
python D:\Essay\scripts\verify_reference.py --doi 10.1109/cvpr.2016.90 --title "Deep Residual Learning for Image Recognition" --author "He" --year 2016 --venue CVPR
```

### 方式二：按标题检索并辅助核验

```powershell
python D:\Essay\scripts\verify_reference.py --title "Deep Residual Learning for Image Recognition" --author "He" --year 2016 --venue CVPR
```

### 方式三：输出 JSON

```powershell
python D:\Essay\scripts\verify_reference.py --doi 10.1109/cvpr.2016.90 --json
```

## 输出如何看
脚本会输出：

- 查询方式
- Crossref 命中的标题、作者、年份、DOI、来源
- 每个字段是“匹配 / 不匹配 / 未提供”
- 总体结论

## 推荐判定标准
### 可入库
以下情况基本可以判定为可入库：

- DOI 命中
- 标题匹配
- 作者基本匹配
- 年份一致
- 期刊 / 会议名称基本一致

### 待人工复核
以下情况要改成“待人工复核”：

- Crossref 没命中
- DOI 命中但标题对不上
- 作者或年份明显不一致
- 会议 / 期刊名称差异过大

### 不建议直接引用
以下情况不要直接进入论文写作：

- 标题和 DOI 对不上
- 作者与年份都明显不一致
- 来源是无法解释的陌生出版物

## 与毕业论文写作的关系
外文文献核验通过后，再做下面两件事：

1. 入 Zotero
2. 写文献事实卡片

不要跳过这一步直接引用，否则后期最容易出现“看起来像真文献，实际信息错了”的问题。

## 建议配套记录
为了避免“脚本跑过了，但后面忘了核验结论”，建议每篇外文文献都补一份记录。

当前仓库已提供模板：

- `D:\Essay\literature\templates\外文文献核验记录模板.md`

推荐做法：

1. 先运行脚本
2. 再把输入信息、Crossref 返回、匹配情况和最终结论记录进模板
3. 只有结论为“可入库”的文献，才继续进入 Zotero 和事实卡片
