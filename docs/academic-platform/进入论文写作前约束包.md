# 进入论文写作前约束包

## 1. 论文写作允许使用的材料范围
后续论文写作时，只允许使用以下材料：

1. 已入 Zotero 的中文文献
2. 已通过核验并已入 Zotero 的外文文献
3. 已完成事实卡片的中文或外文文献
4. 真实项目代码、数据库、页面、接口、测试记录等项目材料

## 2. 论文写作禁止行为
- 不得直接引用未核验的外文文献
- 不得把模型总结当成文献原话
- 不得写入没有页码或出处定位的“事实性判断”
- 不得用临时网页搜索结果替代正式入库文献
- 不得把尚未发生的项目实现写成已完成

## 3. 章节写作顺序建议
推荐顺序：

1. 论文目录与章节目标
2. 绪论所需中文背景文献
3. 相关研究与技术路线文献
4. 项目真实材料整理
5. 需求分析
6. 系统设计与实现
7. 测试与结果分析
8. 结论与展望

不要一开始就写摘要或绪论大段正文，应先把材料收齐。

## 4. 哪些内容必须基于真实项目写
以下内容必须以你自己的项目材料为准：

- 需求分析
- 系统架构
- 数据库设计
- 页面与功能实现
- 接口设计
- 测试过程与测试结果

这些部分不能靠“常规系统应该怎样”去套模板。

## 5. 哪些内容必须基于已核验文献写
以下内容应优先基于已核验文献：

- 研究背景
- 国内外研究现状
- 方法选择依据
- 技术路线的优缺点比较
- 某些标准性定义、行业性事实或已有结论

## 6. 如何避免空话
- 每一段先明确要证明什么
- 每个事实句都尽量对应到 1 个文献事实点或 1 个项目事实点
- 写不出出处的句子，先不写进正文
- 用“问题、做法、结果”三段式代替空泛评价

## 7. 如何避免 AI 味
- 不要用大段抽象套话开头
- 不要连续堆“随着……发展”“具有重要意义”这类空泛句式
- 多写与你项目和文献直接对应的具体内容
- 用真实字段、真实流程、真实实现细节代替模板化表述

## 8. 如何避免假文献
- 中文文献优先来自 CNKI 并完成入库
- 外文文献先走核验脚本
- 对 DOI、作者、年份、来源不一致的文献，不进入正文
- 不凭记忆手打参考文献信息

## 9. 如何避免查重和 AIGC 风险进一步升高
- 不直接复制 PDF 原文或网页原文
- 不连续照搬摘要、引言、结论中的句子
- 先抽事实，再用自己的论文语境重写
- 事实要保留，句式要重组，且必须能回溯出处

## 10. 后续让 Codex 分章写作时应如何喂材料
每次只给 Codex 一个清晰写作任务包，建议包含：

1. 本章标题和目标
2. 可用的文献事实卡片
3. 可用的项目事实材料
4. 禁止引用的材料范围
5. 学校格式要求或字数要求

推荐输入顺序：

1. 先给章节目标
2. 再给 3 到 8 张相关事实卡片
3. 再给对应项目材料
4. 最后再让 Codex 组织成正文草稿

## 11. 进入正文写作前的最低完成标准
达到下面条件后，再进入正式写作最稳：

1. 至少 3 篇中文文献已入 Zotero
2. 至少 2 篇外文文献已完成核验
3. 至少 2 到 3 张事实卡片已完成
4. 项目真实材料已开始整理
5. 已确认哪些章节主要依赖文献，哪些章节主要依赖项目事实
