# 安装与接入说明

## 1. 中文文献主入口：CNKI
本平台把 CNKI 作为中文文献主检索入口，但不绕过任何学校权限、账号权限或网页限制。

推荐做法：

1. 先通过学校图书馆入口、校园网或正规远程访问方式打开 CNKI
2. 搜索与你的软件工程毕业论文题目直接相关的中文文献
3. 优先打开“详情页”而不是搜索列表页
4. 在详情页再决定是否用 Zotero Connector 采集

说明：

- CNKI 是否能下载全文，取决于你的账号和学校权限
- 平台只建立“人工检索 + 合法下载 / 合法入库 + 后续证据整理”的流程

## 2. Zotero Connector 当前状态
这台机器上已经检测到：

- 浏览器：Microsoft Edge
- 扩展：Zotero Connector `5.0.199`

CNKI 支持依据：

- Zotero 官方 translator 仓库存在 `CNKI.js`
- translator 目标域名为 `cnki.net`

这意味着：

- 从能力上看，Zotero 生态确实支持 CNKI 页面元数据采集
- 但是否能在你当前账号会话下成功采集，仍要以真实页面点击结果为准

## 3. CNKI 入库的最短操作
### 优先方式：Zotero Connector
1. 启动 Zotero Desktop
2. 用 Edge 打开 CNKI 的单篇文献详情页
3. 观察浏览器工具栏里的 Zotero Connector 图标是否变成“文章 / 论文 / 书籍”类型图标
4. 点击导入
5. 导入后在 Zotero 中检查题名、作者、年份、期刊、页码是否完整

### 失败时的保底方式：人工入库
如果 Connector 因页面结构、权限、验证码或网络原因没导入成功，不要停住，直接人工入库：

1. 在 Zotero 中新建条目
2. 从 CNKI 详情页手工填写：
   - 题名
   - 作者
   - 年份
   - 期刊 / 学位论文 / 会议名称
   - 卷期页码
   - DOI（若有）
   - URL
3. 若你有合法下载到的 PDF，再把 PDF 作为附件拖入 Zotero

这种方式虽然慢一点，但最稳，也最适合毕业论文这种重视真实性的场景。

## 4. Zotero 集合建议
建议先手动创建这些集合：

- `01-绪论`
- `02-需求分析`
- `03-相关研究`
- `04-技术实现`
- `05-测试与实验`
- `06-结论与展望`
- `99-待判定`

导入文献时，不要只看“是不是相关”，还要立即判断“将来更可能服务哪一章”，否则后续很容易堆很多文献却写不出有证据的正文。

## 5. Better BibTeX
当前 Better BibTeX 已安装并处于激活状态。

建议你后续这样使用：

1. 先保证 Zotero 条目信息真实、完整
2. 再用 Better BibTeX 生成更稳定的 citation key
3. 在论文写作后期统一导出 BibTeX 或在需要时导出 CSL JSON

当前限制：

- 因为库里还没有真实条目，这次没有做“真实文献导出文件”的回归演练
- 平台已经具备导出基础，但最终效果仍建议你在导入第一批文献后做一次实际导出检查

## 6. Doc2X
当前 Doc2X 状态如下：

- 官网可访问
- API 文档可访问
- 开放平台可访问
- Codex MCP 接入文档可访问

但目前没有完成的部分是：

- `DOC2X_API_KEY`
- 带密钥的正式 API / MCP 接入

如果你已经有密钥，后续可按官方方式接入；如果你暂时没有密钥，建议先用网页登录后手动上传 PDF 做结构化解析，再把结果沉淀成事实卡片。

## 7. 什么时候算“接入完成”
### CNKI
以下条件同时满足时，才能算 CNKI 入库链路真正完成：

- 你能正常访问 CNKI
- Zotero Connector 或人工方式至少成功入库 1 篇中文文献
- 入库条目信息经人工核对无明显缺失

### Doc2X
以下条件同时满足时，才能算 Doc2X 接入真正完成：

- 你提供了真实账号登录或 `DOC2X_API_KEY`
- 至少成功解析 1 个真实 PDF
- 解析结果可以落到 Markdown 或事实卡片中
