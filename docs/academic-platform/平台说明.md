# 本科软件工程毕业论文写作平台说明

## 目标
本平台只负责“真实文献获取、管理、核验、证据沉淀”四件事，不直接代写论文正文。

平台面向中国高校软件工程本科毕业论文场景，核心原则是：

- 中文文献以中国知网（CNKI）为主检索入口
- Zotero 是唯一主文献库
- Better BibTeX 负责稳定导出
- 开放来源检索只做补充，不替代 CNKI
- 外文文献先核验真实性，再进入写作
- PDF 先结构化或做事实卡片，再进入后续写作

## 当前平台结构
当前平台按六层组织：

1. 中文文献主检索层：CNKI + Zotero Connector / 人工入库
2. 公开学术检索补充层：`paper-search`
3. 文献管理层：Zotero + Better BibTeX
4. 文献真实性校验层：Crossref 校验脚本
5. PDF 结构化层：Doc2X 网页 / API，当前仅完成入口核验
6. 证据整理层：Markdown 文献事实卡片

## 当前真实状态
### 已可用
- Zotero Desktop
- Better BibTeX for Zotero
- `paper-search` MCP
- Zotero MCP
- 外文文献真实性校验脚本
- 文献事实卡片模板

### 部分可用
- Edge 版 Zotero Connector 已安装，但 CNKI 页面采集仍需你在真实 CNKI 页面上做一次人工点击验证
- Doc2X 官方入口和 API 文档可访问，但未完成带密钥的正式接入

### 不承诺或不伪造的能力
- 不承诺自动抓取 CNKI 全文
- 不承诺在没有学校权限或账号登录的情况下获取受限 PDF
- 不承诺 Doc2X 已完成 API 接入
- 不承诺外文文献只靠标题搜索就一定准确，必须结合 DOI / 作者 / 年份复核

## 目录说明
### `docs/academic-platform/`
存放平台级说明文档。

### `literature/cnki/`
放从 CNKI 获取并确认可信的中文文献附件或整理文件。

### `literature/foreign/`
放外文文献 PDF、导出文件或核验后的辅助材料。

### `literature/notes/`
放每篇文献对应的事实卡片，当前已提供 1 份空白中文卡片样例。

### `literature/templates/`
放模板文件，当前已提供文献事实卡片模板和外文文献核验记录模板。

### `scripts/`
放可运行的小工具，当前已提供外文文献真实性校验脚本。

## 建议的 Zotero 集合结构
建议在 Zotero 中建立以下顶层集合：

- `01-绪论`
- `02-需求分析`
- `03-相关研究`
- `04-技术实现`
- `05-测试与实验`
- `06-结论与展望`
- `99-待判定`

这样做的目的不是提前写章节，而是让每条文献在入库时就被放到“以后要为哪一章服务”的位置，减少后期堆摘要、找不到证据来源的问题。

## 使用边界
这套平台适合做：

- 中文文献检索与入库
- 外文文献补充检索与核验
- 参考文献规范化导出
- 基于真实 PDF 和真实出处的证据沉淀

这套平台暂时不替你完成：

- 自动通过学校权限下载所有 CNKI 全文
- 自动完成 Doc2X API 授权
- 自动生成符合学校格式的最终参考文献表
- 自动通过查重或 AIGC 检测
