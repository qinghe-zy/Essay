# 测试材料包

## 测试环境说明
- 测试对象：前后端分离技术博客推荐系统。
- 前端环境：Vue 3、Vite、Element Plus、Axios、ECharts。
- 后端环境：Java 21、Spring Boot 4.0.3、MyBatis、MySQL。
- 数据基础：`blog_system.sql` 中的 `user`、`blog`、`comment`、`notification`、`user_like`、`user_action`、`visit_log` 等表。
- 当前说明：本轮不做性能测试；已补充前端构建验证、后端 `contextLoads()` 测试和接口级联调记录，但尚未形成完整前端页面截图与端到端联调证据。

## 功能测试项
| 测试项 | 测试内容 | 预期结果写法 |
|---|---|---|
| 注册 | 新用户注册 | 系统能够完成注册并返回成功提示，重复用户名会被拦截 |
| 登录 | 正确和错误凭据登录 | 正确凭据可进入系统，错误凭据会被拦截 |
| 资料修改 | 昵称、头像、密码修改 | 用户资料可保存，修改密码后需重新登录 |
| 文章发布 | 标题、正文、标签、封面上传 | 文章可成功保存并出现在列表中 |
| 文章编辑 | 修改文章内容 | 修改结果可在详情页重新展示 |
| 文章删除 | 作者或管理员删除 | 有权限用户可删除，无权限用户被拦截 |
| 搜索 | 按标题或标签检索 | 系统返回匹配文章列表 |
| 点赞 | 点赞与取消点赞 | 点赞状态和计数能正确切换 |
| 收藏/待读/拉黑 | 三类状态切换 | 状态能写入并在个人中心正确回显 |
| 评论与评分 | 提交评论、只评分、删除评论 | 评论可展示，评分影响文章平均分，删除后均分重算 |
| 消息通知 | 点赞和评论后查看消息 | 可看到消息、未读数变化、消息可设为已读 |
| 浏览历史 | 阅读文章并离开页面 | 浏览记录和阅读时长能更新 |
| 推荐页 | 打开推荐页并切换“换一批” | 推荐列表可加载，候选池可分批切换 |

## 关键业务流程测试项
| 流程 | 步骤摘要 | 预期结果写法 |
|---|---|---|
| 注册到登录 | 注册后立即登录 | 用户可完成从注册到登录的完整闭环 |
| 发布到浏览 | 发布文章后在首页或搜索中找到并打开 | 新发布文章可被检索并正常展示 |
| 浏览到画像 | 阅读文章并停留后进入个人中心 | 阅读行为可体现在历史记录、时长统计或画像摘要中 |
| 互动到通知 | 用户 A 点赞或评论用户 B 的文章 | 用户 B 可在消息中心看到对应提醒 |
| 评分到均分 | 用户评分后刷新详情 | 文章综合评分能按预期更新 |
| 拉黑到过滤 | 拉黑文章后重新查看列表或推荐 | 被拉黑内容不再进入相关列表或推荐 |

## 异常输入与边界测试项
| 测试项 | 场景 | 预期结果写法 |
|---|---|---|
| 错误登录 | 用户不存在或密码错误 | 系统返回失败提示，不允许进入系统 |
| 未登录访问 | 直接访问受限路由 | 路由守卫将用户重定向到登录页 |
| 空文章发布 | 标题或正文为空 | 系统提示不能为空，不提交数据 |
| 标签缺失 | 发布文章未输入标签 | 系统提示至少输入一个标签 |
| 短文本 AI 分析 | 正文长度不足 | 系统提示内容不足，或不执行分析 |
| 无效阅读时长 | 秒数为空或小于等于0 | 后端返回无效时长，不更新数据 |
| 空评论空评分 | 评论内容和评分同时为空 | 前端拦截，不提交请求 |
| 无权限删除 | 非作者且非管理员删除文章或评论 | 系统返回权限不足或无权删除 |

## 推荐逻辑定性验证项
| 验证项 | 观察点 | 预期结果写法 |
|---|---|---|
| 冷启动推荐 | 行为较少用户打开推荐页 | 系统以热门内容为主，并显示冷启动解释 |
| 非冷启动推荐 | 有浏览、点赞、收藏记录后查看推荐 | 推荐结果开始体现兴趣标签和历史偏好 |
| 推荐理由展示 | 推荐列表查看 `recommendReason` | 推荐结果可显示与兴趣相关的解释语句 |
| 黑名单过滤 | 拉黑文章后再次请求推荐 | 被拉黑内容不再进入推荐结果 |
| 候选池切换 | 连续点击“换一批推荐” | 系统可按候选池批次切换推荐内容 |
| 相关推荐 | 详情页查看相关推荐 | 系统优先展示同标签内容，不足时回退热门内容 |

## 可直接写入论文的结果写法
- 可以写“经人工联调验证，该功能能够完成预期业务流程”。
- 可以写“测试结果表明，该页面交互与后端返回逻辑基本一致”。
- 可以写“在冷启动场景下，系统能够回退到热门内容推荐，保证推荐页可用性”。
- 可以写“本轮测试在本地数据库已存在基础数据的环境下完成，接口返回与数据库回查结果能够相互印证”。

## 明确禁止的结果写法
- 不写推荐准确率、召回率、NDCG、MAP、AUC、F1。
- 不写 TPS、QPS、吞吐量、平均响应时间。
- 不写“性能显著提升”“推荐效果提升 xx%”等量化结论。
